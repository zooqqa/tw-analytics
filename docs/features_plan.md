# Планы развития после MVP

## Этап 0: Доработка MVP (1-2 недели)

### Инкрементальная загрузка
- Замена полного удаления на upsert по диапазону дат
- Сохранение истории загрузок

### UI улучшения
- История загрузок в интерфейсе
- Пресеты фильтров (сохранение/загрузка)
- Экспорт данных (CSV/JSON/Excel)
- Toast-уведомления об ошибках

### Тесты
- Unit тесты для загрузчиков и метрик
- Integration тесты для API
- Фикстуры с тестовыми данными

## Этап 1: Управление офферами (2-3 недели)

### CRUD офферов
```sql
offers (name, product, partner_program, geo, rate, cap_daily, kpi_requirement, status)
```

### Мониторинг
- Отслеживание остатков капов (70%, 90%, 100%)
- Алерты по ROI < 0%, отсутствию конверсий
- Уведомления в UI/Telegram

## Этап 2: API интеграции (3-4 недели)

### Коннекторы
- TrafficWave API (ежечасная синхронизация)
- Moloco API (данные о тратах)
- Планировщик задач (APScheduler)

### Автоматизация
- Синхронизация по расписанию
- Retry логика при сбоях
- Мониторинг состояния интеграций

## Этап 3: Аналитика (4+ недель)

### Прогнозирование
- Планирование недельных бюджетов
- Тренды и сезонность
- Рекомендации по оптимизации

### Масштабирование
- Поддержка Facebook/Google Ads
- Настраиваемые дашборды
- Интеграция с внешними BI

## Приоритеты

**Критично**: Инкрементальная загрузка, тесты, обработка ошибок
**Важно**: История загрузок, экспорт, пресеты фильтров
**Перспективно**: API интеграции, ML рекомендации
